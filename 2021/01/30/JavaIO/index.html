重载运算符

-
运算对象至少有一个类类型

-
不能改变运算符优先级和结合型

-
有些不能重载，如  ::  .   .*   ?:

-
MEC++ Item 7 ：永远不要重载&&，||和  ,

-
运算对象求值顺序是确定的（ ? : 也是）

-
重载后运算对象变为函数的参数，函数参数的求值顺序是未定义的

-
不是成员函数的话依次绑定到参数上，成员函数第一个对象绑定到this上

不允许两个c风格字符串相加，两个内置类型不能重载运算符

IO库

左移运算符<<有左结合

输入要考虑输入失败的情况

  • 不抛出异常，但有方法检测
  • 放在if()，显示向bool转换，探测是否正常

读到不是int的会返回false

Manipulators操纵符

-
endl输出换行符同时刷新输出流的缓冲区

  • 一个一个字符输出效率较低

  • 通常攒几个字符到缓冲区再定期刷新缓冲区往外写

  • flush刷新缓冲区

  • boolalpha，noboolalpha C++ Primer 17.5

重载<< ,>>,getline等用引用 向上转型

子类中包含完整的父类

-
多重继承导致iostring同时包含两个basic_ios

-
使用虚拟继承可以处理

写i/ostream时不知道后面有人要多重继承，没写虚拟继承。但iostream自己需要多重继承却要求i/ostream设计者用虚拟继承

File mode文件模式  用| 组合

标准库为什么不提供矩阵

-
基于现有的很容易做出的，标准库不会提供

-
具体操作的优化有很多种选择

istringstream

ostringstream

标准库

-
尽可能避免底层的容易出错的操作

  • 自己定义字符数组里面的元素是未定义的，开string容器不会是未定义
  • stl提供的都是值语意

Operation

s1+=s2 和s1=s1+s2不一样

+=类似push_back

计算s1+s2产生新的string对象，需要拷贝一次s1原有的内容

c11以前赋值也需要拷贝一次，c11后右值可以移动

stl所有容器除了forward_list都有.size()

-

-
模板参数是类型的一部分

  • 所有类型都需要在编译时知道
  • 模板参数还可以是在编译时确定的整数：常量表达式

Vector

花括号初始化

-
用花括号调用构造函数在大多数情况下都适用

-
显示调用默认构造函数时若用（）编译器会认为你在声明函数

-
类内初始值

-
vector允许用花括号接受初始化列表（不是初始值列表）是败笔

-
初始化列表有很高优先级

-
给花括号调用构造函数的初始化方式的推广造成困扰

-
空花括号一定表示默认初始化，而不是把空的列表传给接收列表的构造函数

vector的比较运算符依赖于vector里元素类型的<运算符

  • 尽可能减少对未知类型的要求，有<就可以实现4个

vector不提供operator+的重载

-
有连接或者对应元素相加多重语意

-
避免重载运算符的滥用

下标访问

-
[]下标运算符不检查越界

-
有下标访问的容器也会提供 .at(), 会检查边界，越界会抛出异常

不要在基于范围的for语句中修改容器的大小

迭代器

-
it的类型为

-
v.empty(), v.end() 返回尾后（off-the-end）迭代器，左闭右开区间

  • 相减得到元素个数
  • begin始终<=end
  • 空容器：v.begin()=v.end()

迭代器支持的操作

-

  • 尾后迭代器不指向实际元素

stl很多组件对错误行为 静默，如传越界值不检查，最后导致segmentation fault

  • 与内置数组保持一致

Range-for原理

v需要满足的条件

  • 有begin，end，返回广义的迭代器

    • 迭代器支持解引用，++，!=（++是前置的，不用拷贝自己）

const_iterator

  • const对象上的begin()，end()返回const的iterator

指向不同容器的迭代器的比较返回未定义的结果

迭代器相减得到带符号整数类型difference_type

不是所有迭代器都提供< , 所有迭代器都提供== !=
